# Fix: Index Mismatch (HTML 1-indexed vs JS 0-indexed)

## Problem

HTML uses `data-position="1", "2", "3", "4"` (1-indexed) but JavaScript uses 0-3 array indices (0-indexed). This causes:
- INIT command: Wrong player nodes updated (4th position shows wrong format)
- SETTLE command: Player indices off by one, advancement paths incorrect
- RESET command: 4th element from each group can't be reset
- Player states not applied to correct nodes

## Root Cause

In `index.html`:
```html
<div class="player-node" data-group="A" data-position="1"> <!-- 1-indexed -->
```

In `app.js`:
```javascript
playerNames.forEach((name, position) => {  // position is 0, 1, 2, 3
    this.updatePlayerNode(groupName, position, ...);  // Looks for data-position="0"
});
```

The selector `[data-group="A"][data-position="0"]` finds nothing because HTML has `data-position="1"`.

## Solution

Change HTML to use 0-indexed positions (0, 1, 2, 3) to match JavaScript array indices.

## Files to Modify

### 1. index.html

Change ALL `data-position` attributes from 1-4 to 0-3:

- Group A: data-position="1"→"0", "2"→"1", "3"→"2", "4"→"3"
- Group B: data-position="1"→"0", "2"→"1", "3"→"2", "4"→"3"
- Group C: data-position="1"→"0", "2"→"1", "3"→"2", "4"→"3"
- Group D: data-position="1"→"0", "2"→"1", "3"→"2", "4"→"3"
- Group AB: data-position="1"→"0", "2"→"1", "3"→"2", "4"→"3"
- Group CD: data-position="1"→"0", "2"→"1", "3"→"2", "4"→"3"
- Finals: data-position="1"→"0", "2"→"1", "3"→"2", "4"→"3"

### 2. app.js

No changes needed - the code already uses 0-indexed positions correctly.
