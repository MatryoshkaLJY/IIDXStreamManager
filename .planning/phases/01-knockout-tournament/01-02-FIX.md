# Fix: Finals Group Name Mismatch

## Problem

Semifinal to final advancement doesn't work. The top 2 players from AB and CD don't advance to finals positions.

## Root Cause

In `index.html`, finals nodes use `data-group="F"`:
```html
<div class="player-node final" id="F1" data-group="F" data-position="1">
```

But in `app.js`, the code uses `finals` and `finals-X` identifiers:
```javascript
// In handleSettle for semifinal:
const fromId = `${group}-${player.position}`;  // "AB-0"
const toId = `finals-${targetPosition}`;       // "finals-0"
this.lightPath(fromId, toId);
this.updatePlayerNode('finals', targetPosition, ...);

// In getPlayerNode:
return document.querySelector(`[data-group="${group}"][data-position="${position}"]`);
// Looking for: [data-group="finals"][data-position="0"] - NOT FOUND
```

The mismatch means:
- `updatePlayerNode('finals', 0, ...)` looks for `[data-group="finals"]` but HTML has `[data-group="F"]`
- `lightPath('AB-0', 'finals-0')` looks for path with `data-path="AB-0-finals-0"` but paths use `F`

## Solution

Change `index.html` finals nodes from `data-group="F"` to `data-group="finals"` to match JavaScript.

## Files to Modify

### index.html

Change finals section:
```html
<!-- Before -->
<div class="player-node final" id="F1" data-group="F" data-position="1">

<!-- After -->
<div class="player-node final" id="F1" data-group="finals" data-position="0">
```

Update all 4 finals nodes:
- F1: data-group="F"→"finals", data-position="1"→"0"
- F2: data-group="F"→"finals", data-position="2"→"1"
- F3: data-group="F"→"finals", data-position="3"→"2"
- F4: data-group="F"→"finals", data-position="4"→"3"

Also update path data-path attributes that reference finals:
- path-AB1-F1, path-AB2-F2, etc. should use finals-0, finals-1, etc.
