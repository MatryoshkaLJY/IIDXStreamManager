# Fix: SCORE Display Styling

## Problem

User reported: "player-score should be small text, player-points should be large text"

Currently the styling is either reversed or same size.

## Root Cause

CSS styling for `.player-score` and `.player-points` needs adjustment.

## Solution

Update CSS in `style.css`:

```css
.player-score {
    font-size: 0.75rem;  /* Small text */
    opacity: 0.8;
    /* Keep other existing styles */
}

.player-points {
    font-size: 1.25rem;  /* Large text, prominent */
    font-weight: 700;
    color: #00ffff;      /* Cyan accent for visibility */
    /* Keep other existing styles */
}
```

## Verification Steps

1. Send INIT to populate player names
2. Send SCORE command:
   ```json
   {"cmd": "score", "data": {"stage": "quarterfinal", "group": "A", "round": 1, "scores": [{"player": "P1", "score": 990000}, {"player": "P2", "score": 980000}, {"player": "P3", "score": 970000}, {"player": "P4", "score": 960000}]}}
   ```
3. Verify score displays small (e.g., "990,000")
4. Verify points displays large (e.g., "2 pts")
5. Check visual hierarchy is clear

## Success Criteria

- [ ] Raw score displayed in smaller text
- [ ] Points displayed in larger, prominent text
- [ ] Visual hierarchy clearly distinguishes score vs points
- [ ] Styling matches BPL aesthetic (cyan accent for important data)
