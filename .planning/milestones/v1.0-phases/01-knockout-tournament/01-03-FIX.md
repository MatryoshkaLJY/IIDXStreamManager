# Fix: Visual Layout Issues

## Problem

User reported:
1. Layout is crowded
2. Player nodes are too large and overlapping
3. Group B and D labels are at the bottom instead of proper positions
4. SVG connection lines look messy
5. User explicitly stated: "I don't need these extra connection lines"

## Root Cause

1. Player node dimensions too large for 1920x1080 with 16 nodes
2. CSS positioning causes overlap
3. Group label positioning incorrect for B/D
4. Too many SVG paths create visual clutter

## Solution

### Option A: Remove SVG Paths Completely (Recommended)

User explicitly said they don't want the connection lines. Remove all SVG paths and simplify the design.

### Option B: Fix Layout with Cleaner Paths

If paths are needed, reduce node sizes and reposition for better spacing.

## Implementation: Option A (Remove Paths)

### 1. index.html

Remove the entire SVG section:
```html
<!-- Remove this entire block -->
<svg class="connection-svg" viewBox="0 0 1920 1080" preserveAspectRatio="xMidYMid slice">
    ... all path elements ...
</svg>
```

### 3. app.js

Remove path lighting functions or make them no-ops:

```javascript
lightPath(from, to, isChampion = false) {
    // No-op: user doesn't want connection lines
    return;
}

clearAllPaths() {
    // No-op

    const championEl = document.querySelector('.champion-name');
    if (championEl) {
        championEl.textContent = champion.name;
    }
}
```
