## Phase 6 Specific Contracts

### Review Panel Placement
- Location: Expandable panel inside `templates/status.html`, below the Scene Switching card.
- Behavior: Auto-expands when `pending_scores` is non-empty; operator can manually collapse it.
- Collapse control: A small header button (e.g., "Hide" / "Show") on the panel's title bar.

### Pending Score Rows
- Each cabinet with a pending score gets one row in the panel.
- Row layout (left to right):
  1. Cabinet label (e.g., IIDX#1)
  2. Player name(s) assigned to that cabinet
  3. 1P score (editable inline)
  4. 2P score (editable inline)
  5. Valid/invalid indicator per score
- Inline editing: Clicking a score value turns it into a number input. Blur or Enter commits the edit.

### Invalid Score Highlighting
- Scores with `1p_valid === false` or `2p_valid === false` are shown with:
  - Text color: `var(--color-warning)` (#f59e0b)
  - Left border: 3px solid `var(--color-warning)` on the score cell
  - A small "!" prefix before the value (or in the cell) for quick scanning

### Confirm & Push Button
- Disabled state: Button is `disabled` and visually dimmed (opacity 0.5) until all invalid scores are corrected.
- Enabled state: Full accent background (#22c55e), white text, pointer cursor.
- On click: Immediately switches OBS to `Scoreboard_web`, then starts the countdown timer.

### Countdown Timer
- Displayed inside the review panel as a prominent label: "Pushing in {N}s..."
- Visual: Large numeric countdown (20px, weight 700) centered in the panel.
- At 0: Label changes to "Pushing scores..." with a pulsing opacity animation.
- After push succeeds: Panel collapses and pending scores are cleared.

### Scoreboard Delay Control
- A small inline form on the Status page, inside the review panel header or just above it.
- Label: "Scoreboard delay (s)"
- Input: number stepper, min 0, step 0.5, default 5.0
- Save: On blur or Enter, POST to `/api/scoreboard_delay` (or similar) and update `RuntimeState`.

### Multiple Simultaneous Scores
- If 2+ cabinets have pending scores, all rows are rendered in the same panel.
- The single "Confirm & Push" button applies to the entire batch.
- Rows are sorted by cabinet number (IIDX#1 → IIDX#4).
