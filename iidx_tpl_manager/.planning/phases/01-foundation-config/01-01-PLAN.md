## Schema Specifications (from 01-CONTEXT.md D-01, D-02, D-03)

### teams.json
```json
{
  "teams": [
    {
      "id": "string",
      "name": "string",
      "emoji": "string",
      "colors": {
        "primary": "string",
        "secondary": "string",
        "accent": "string"
      },
      "players": [
        {
          "id": "string",
          "name": "string",
          "role": "string"
        }
      ]
    }
  ]
}
```

### team_schedule.json
```json
{
  "weeks": [
    {
      "matches": [
        {
          "left_team": "string",
          "right_team": "string",
          "template": "string",
          "rounds": [
            {
              "type": "1v1|2v2",
              "theme": "string",
              "left_players": ["string"],
              "right_players": ["string"]
            }
          ]
        }
      ]
    }
  ]
}
```

### individual_schedule.json
```json
{
  "groups": {
    "Group A": ["player1", "player2", "player3", "player4"],
    "Group B": ["player1", "player2", "player3", "player4"]
  }
}
```
Note: group names are dynamic keys; the model should use `Dict[str, List[str]]`.

## UI Copy Contract (from 01-UI-SPEC.md)

- Empty state heading: "No config loaded"

- Generated templates must contain valid minimal structure (empty arrays/objects) so they pass validation.
- If an existing config file has schema errors, block startup and emit a clear terminal/web error with actionable detail.
