## Specific implementation notes (from 02-CONTEXT.md)
- PROG-01 E/F composition:
  - Group A 1st → E1, Group A 2nd → F1
  - Group B 1st → F2, Group B 2nd → E2
  - Group C 1st → F3, Group C 2nd → E3
  - Group D 1st → E4, Group D 2nd → F4
- Order: A → B → C → D → E → F → Finals
- PROG-02: In `handleSettle`, after processing a non-final group's settlement, automatically call `handleContinue()` to advance the active highlight.
- SCOR-02 sorting:
  - Groups A-F: sort by PT descending, then total raw score descending.
  - Finals: sort by PT descending only. If tied, enter a tiebreaker round where tiebreaker scores accumulate and are used to break the tie.
- UI-02: Re-use the existing `medals` array in `handleSettle` for finals. `updatePlayerNode` sets `.player-points` to the medal emoji. Champion gets `.champion` class (gold glow already defined in CSS). 2nd place gets silver flash, 3rd gets bronze flash.
