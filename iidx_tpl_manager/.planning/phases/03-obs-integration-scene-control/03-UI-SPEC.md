## Phase-Specific UI Elements

### OBS Connection Config Form
- Inline on the status page (per RESEARCH.md recommendation)
- Fields: Host (text), Port (number), Password (password, optional)
- Submit button label: "Save & Reconnect"
- Form uses `.card` container with `.heading` title "OBS Connection"

### Scene Switching Controls
- Four buttons arranged horizontally below the config form
- Button labels match scene names: `现场摄像`, `SP团队赛`, `DP团队赛`, `个人赛`
- Disabled state (opacity 0.5, cursor not-allowed) when OBS is disconnected or scenes are invalid
- Each button submits via HTTP POST to `/switch_scene`

### Warning Banner
- Full-width banner inside `.card` at top of status page when OBS is disconnected or scenes are missing
- Uses `.status-error` text color on secondary background
- Includes inline "Reconnect to OBS" or "Retry Validation" button
- Banner is hidden when connection and validation are healthy

### Connection Status Indicator
- Small label below the banner: "OBS: Connected" (`.status-ok`) or "OBS: Disconnected" (`.status-error`)
- Updated in real time via SocketIO `obs_status` events
