<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Verify OBS integration end-to-end</name>
  <files></files>
  <action>Pause execution and wait for the human to verify the integrated OBS control panel using the steps below.</action>
  <what-built>
    - Backend: RuntimeState with OBS config, lazy-connect OBSClient, SceneController with validation, background heartbeat, Flask routes, and SocketIO handlers
    - Frontend: Status page with OBS config form, four scene-switching buttons, real-time warning banner, and SocketIO-powered operator.js
  </what-built>
  <how-to-verify>
    1. Start OBS Studio locally and ensure WebSocket v5 is enabled on port 4455 (default).
    2. Create the four required scenes in OBS if they do not exist: 现场摄像, SP团队赛, DP团队赛, 个人赛.
    3. Start the Flask app: `python src/app.py`
    4. Open http://localhost:5002 in a browser.
    5. Verify startup resilience:
       - If OBS is running and scenes exist, the page should show "OBS: Connected" and the scene buttons should be enabled.
       - If any scene is missing, a warning banner should appear listing the missing scenes, and buttons should be disabled.
    6. Verify scene switching:
       - Click each of the four scene buttons and confirm OBS switches to the corresponding scene.
    7. Verify disconnect handling:
       - Stop OBS Studio. Within ~3-6 seconds, the banner should appear with "OBS disconnected. Scene switching and auto-transitions are paused."
       - The scene buttons should become disabled.
    8. Verify manual reconnect:
       - Restart OBS Studio.
       - Click "Reconnect to OBS" in the banner.
       - The banner should disappear, "OBS: Connected" should appear, and buttons should re-enable.
    9. Verify config form:
       - Change the port to an invalid value (e.g., 99999) and click "Save & Reconnect".
       - The UI should remain stable and the connection should fail gracefully (no crash).
  </how-to-verify>
