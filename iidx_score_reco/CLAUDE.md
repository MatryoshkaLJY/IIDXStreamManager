# IIDX Score Recognition 项目进度

## 项目目标
从 IIDX 游戏结算画面截图中自动识别 SCORE 数字。

## 当前进度

### 已完成

1. **模板加载器** (`recognizer.py`)
   - 支持多尺度模板匹配 (scales: 0.9, 1.0, 1.1)
   - 自动提取白色像素掩码
   - 边界裁剪优化
   - 垂直投影法分割连续数字

2. **核心识别流程**
   ```
   截图 → ROI裁剪 → 白色掩码提取 → 数字分割 → 多尺度模板匹配 → 结果
   ```

3. **测试脚本** (`test.py`)
   - 支持单图识别
   - 支持 ROI 区域指定
   - 模板自匹配测试

### 发现的关键问题

**模板与实际截图不匹配**

| 对比项 | 模板 (font/*.png) | 实际截图 (data/*) |
|--------|------------------|------------------|
| 样式 | 纯白数字 | 发光渐变数字 |
| 描边 | 黑色描边 | 光晕效果 |
| 示例 | `1` 是简单竖线 | `1` 有发光和装饰 |

**导致结果**: 模板匹配置信度极低 (< 0.1)，无法正确识别。

### 测试数据

- **模板尺寸**: 基础 30x20，多尺度 (27x18 ~ 33x22)
- **截图中数字位置**: 右侧 SCORE 区域约 (920, 340) 附近
- **截图分辨率**: 多种 (960x817, 1200x817)

## 下一步方案

### 方案A: 重新截取模板（推荐）

从 `data/` 中的截图直接截取数字作为新模板：

```bash
# 从截图中手动截取 0-9 的数字样本
# 例如从 54fb29852f3baeac43747e4b108d5043.webp 截取 SCORE "1626" 中的数字
```

需要执行：
1. 查看 `data/` 中的截图，找到清晰的数字
2. 精确定位单个数字的 ROI
3. 截取并保存为新的模板替换 `font/` 目录

### 方案B: 预处理发光效果

尝试用图像处理减弱发光影响：
- 提高白色提取阈值
- 形态学操作 (开运算/闭运算)
- 边缘检测替代阈值分割
